# J.Oli Origins Documentation Hub

This folder is the single source of truth for long-term collaboration on `joliorigins.cz`.

## Core Documents
- [Master Specification](./MASTER_SPECIFICATION.md): complete target state for full website functionality.
- [Collaboration Protocol](./COLLABORATION_PROTOCOL.md): rules for multi-account teamwork, handover quality, naming, and updates.
- [Handover Log](./HANDOVER_LOG.md): chronological implementation log for continuity between sessions and accounts.
- [90-Day Pilot Workflow Summary](./PILOT_90D_WORKFLOW_SUMMARY.md): condensed operational model from the extended implementation brief.

## Required Documentation Workflow
1. Before starting a larger implementation block, review:
   - `MASTER_SPECIFICATION.md`
   - latest entries in `HANDOVER_LOG.md`
2. After every meaningful implementation step, append a new log entry to `HANDOVER_LOG.md`.
3. If scope, architecture, or priorities change, update `MASTER_SPECIFICATION.md` and note the change in `HANDOVER_LOG.md`.
4. Do not rely on chat history as primary project memory. Always write durable updates into these docs.

## Ownership Model
- Product vision and acceptance criteria are anchored in `MASTER_SPECIFICATION.md`.
- Process and team quality are anchored in `COLLABORATION_PROTOCOL.md`.
- Day-to-day continuity is anchored in `HANDOVER_LOG.md`.
